About the Project

Our Solar System project is an animated simulation that visualizes the eight planets orbiting the Sun — with attention to relative orbital speeds, planetary sizes, and the characteristic colors and textures of each world. The goal was to create something both scientifically informed and aesthetically captivating — a simulation that teaches while it delights.

Behind the Project

Tools & Software

Every render, animation, and interaction in this project was built using the following tools.

🧊
Blender
3D Modeling & Animation

Used for all 3D planet modeling, texturing, orbital animations, and final render output. Cycles renderer was used for physically-based lighting.

This interactive showcase page was hand-coded — the planet cards, orbital calculator, and all animations were built without any external frameworks.

🖼️

Reference & Assets

High-resolution planetary surface maps sourced from a publicly available Solar System texture library were used as reference and base textures for every planet.

All web code was written and organized in Visual Studio Code, using the Live Server extension to preview changes in real time during development.

The Sun
Star · Center of the Solar System

The Sun is the gravitational anchor of our entire solar system — a massive ball of hot plasma generating energy through nuclear fusion at its core. It accounts for 99.86% of the solar system's total mass and its gravity keeps all eight planets locked in their orbital paths.

  • Diameter1,392,700 km
  • Surface Temp5,500 °C
  • Dist. from Earth149.6 M km
  • Age~4.6 billion yrs

Process Guide

How We Built It in Blender

A step-by-step walkthrough of how this solar system was modeled, textured, and animated.

01
Scene Setup & Scale

Open Blender and set up your scene units. We used a simplified scale where 1 Blender unit = 1 million km to keep all planets manageable in the viewport. Set the render engine to Cycles and configure HDRI world lighting using a deep space environment map for ambient illumination.

Scene · Lighting · Units
Add your Blender screenshot here — step-01-scene-setup.png
02
Planet Modeling

Add a UV Sphere for each planet (Shift+A → Mesh → UV Sphere). Set the segment count to 64×64 for smooth curvature. Scale each sphere proportionally based on real diameter ratios — for example, Jupiter should be roughly 11× the diameter of Earth. Name each object clearly in the Outliner.

Modeling · UV Sphere · Scale
Add your Blender screenshot here — step-02-planet-modeling.png
03
Texturing with Shader Nodes

Apply a Principled BSDF material to each planet. Import your texture maps from the library and plug them into the Base Color, Roughness, and Normal inputs. For the Sun, use an Emission shader with an orange-white gradient to simulate its glow. Add a Point Light inside the Sun object for scene illumination.

Shading · Nodes · Textures
Add your Blender screenshot here — step-03-shader-nodes.png
04
Orbital Paths & Animation

Use Empty objects as orbit centers. Parent each planet to its corresponding Empty, then animate the Empty's Z-rotation from 0° to 360° across the timeline. Use the orbital period ratios to set relative keyframe counts — Mercury's orbit should complete roughly 4× faster than Earth's in your animation. Enable the Graph Editor to set linear interpolation on all rotation curves.

Animation · Keyframes · Empties
Add your Blender screenshot here — step-04-animation.png
05
Saturn's Ring System

Create a flat disc (Add → Mesh → Circle, then Extrude and scale inward to form a ring shape). Apply an alpha-transparent texture for the ice particle look using a Principled BSDF with the Transmission slider. Tilt the ring object at ~27° to match Saturn's axial tilt. Parent the ring to Saturn's planet object so it rotates together.

Modeling · Transparency · Rings
Add your Blender screenshot here — step-05-saturn-rings.png
06
Camera, Rendering & Export

Position your camera using a wide-angle lens (18–24mm) for the full system overview, or animate it for a dramatic flythrough. In Render Properties, set resolution to 1920×1080 and enable Motion Blur for realism. Render to an image sequence (PNG frames) rather than directly to video — this lets you recover from crashes. Assemble the final sequence in Blender's Video Sequence Editor to export the finished MP4.
